what is Git?

Git is a version control system developed by Linus Torvalds who also created Linux.
A version control system is a software tool that help to track and manage changes to files, code, documents, and any other types of digital content over time.
In software engineering, git helps to track and manage changes to code.
It is also used to collaborate in code by several software engineers.
other version control systems include;

  • mercurial(HG)
  • Apache subversion(SVN)
  • Performance Helix core
  • Bazaar(BZR)
In this blog we will however be focusing on Git only.

How to install Git

For you to use Git you must first install it in your machine, or check if it has been installed.
open the command prompt/ terminal depending on the operating system you are using(windows,linux,macos).
Type the command (git --verion) and hit enter.
if you have Git installed then the response will be the version of Git installed in your machine eg.(git version 2.43.o.windows.1).
if not, then you will see an error message that the command is not recognized.
Go ahead and download Git. Here is a link Git

Configuring Git

After installing Git let it know who you are by passing in your username and email address.
To configure your username use the command (git config --global user.name"your username")
To configure your email address use the command (git config --user.email"your emailaddress")
It is advisable that you use your GitHub email address.

Initializing a Git repository

To use Git in your project, you need to initialize a git repository inside that project.
A git repository is a storage location where GIT version control system manages and stores projects files and revision history.
First check if the project contains a repository to avoid reinitializing by running the command (git status)
If the project does not contain a Git repository you will get the message "fatal not a git repository".
Any other message means that the project contains a repository.
If the project does not contain a git repository you can create/initialize one by using the command (git init)
you will see the text "initialized an empty git repository"if the repository is initialized successfully.

Adding files to the staging area

After creating an empty git repository, you need to add some files.
Use the command git status to verify that the file is part of the repository.
You will see that your file is under untracked files, thus you need to add it to your repository.
Files in your GitHub repository can be in one of the two states: tracked or untracked.
Tracked files: are files that git know about and are added to the repository.
Untracked files: this files are in the working directory they however avent been added to the repository.
A staging environment/area/index is an area where changes to files are prepared before they are committed to repository.
To add a file to the staging environment, use the command (git add filename)
If you have more than one file you can specify the file names (git add file1 file2 file3)
If you want to add all the files in the project instead of specifying those files, use a period symbol (git add .)

Commiting changes

Adding commits keeps track of our progress changes as we work.
Git considers each commit change point or a save point, it is a point we can go back to in our project if we find a bug or we want to make a change.
We use the command (git commit -m"commit message") to commit changes to our repository.
The commit command performs a commit and the -m "message" adds a message.
To view past commits you use the command (git log) or (git log --online) if you want a bit more compressed output of git log.

Git branches

A branch is a new/separate version of the main repository. They allow multiple developers to work on differnet features, bug fixes,
or experiments simultaneously without interfering with each other's work.
when the work is complete, a branch can be merged into the main project

Creating a branch

To create a branch, we use the command (git branch new-branch-name).
You can view all the branches we have using the command (git branch).

Merging a branch

To merge a branch add all the changes made in the branch to the staging area then commit those changes.
After that move to your master/main branch?the branch you want to merge this branch with.
The master/main branch contains the latest, stable version of the codebase, where all finished and tested features are intergrated.
You use the command (git merge branch-name) to merge the branch.

Switching branches

You can change/switch branches using the command (git checkout branch-name)

Deleting brances

To delete a branch you use the command (git btanch -D branch name)
